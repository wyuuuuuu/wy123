# 智能网联汽车教材知识图谱展示系统操作说明

## 1. 项目定位

本文件夹是基于已有教材关系抽取结果构建的知识图谱展示与教学辅助原型。它不训练模型，不修改 Step0-Step6 关系抽取主线，也不扩展成完整在线教学平台。

当前系统主要用于毕业设计答辩展示，重点说明“教材关系抽取结果如何被组织成知识图谱，并进一步服务于教师教学分析和学生学习辅助”。展示层属于应用验证部分，核心研究主线仍然是教材关系抽取与三元组构建。

主要用途：

- 按教材章节组织四个主题集群
- 在图谱总览中展示知识点、关系、章节主题和教材证据规模
- 在教师端展示章节知识结构、核心知识点、关系统计和三元组证据
- 在学生端展示知识点查询、学习进度、图谱问答、学习路径和轻量资源推荐

## 2. 运行环境

建议环境：

- Windows 10 / 11
- Python 3.10 或 Python 3.11

依赖库在 `requirements.txt` 中：

- `pandas`
- `networkx`
- `streamlit`

学生端的 AI 问答和 AI 推荐说明使用本地 Ollama：

- 默认模型：`qwen3:1.7b`
- 默认地址：`http://localhost:11434`
- 使用前请先启动 Ollama，并确保本地已有该模型。

资源推荐使用项目内预置资源库和知识图谱相邻关系进行匹配。大模型只负责生成推荐说明和问答表达，不临时编造网址。

## 3. 第一次运行

先双击：

```bat
01_install_dependencies.bat
```

然后双击：

```bat
03_build_topics_and_start_teaching_app.bat
```

脚本会先生成主题集群，然后启动 Streamlit 页面。浏览器打开终端中显示的本地地址，一般是：

```text
http://localhost:8501
```

## 4. 输入数据

本包已经带有最终审核三元组：

```text
inputs/triples_all_reviewed.json
```

也带有实体词典：

```text
data/dictionaries/dictionary.csv
```

如果需要从三元组重新构建节点表和边表，双击：

```bat
02_rebuild_kg_from_triples.bat
```

然后再运行：

```bat
03_build_topics_and_start_teaching_app.bat
```

## 5. 页面说明

当前界面保留三个主入口：`图谱总览`、`教师端`、`学生端`。

### 图谱总览

图谱总览用于展示教材知识图谱整体结构和基础统计信息，包括知识点总数、实体节点数、关系总数、章节数和资源证据数。中间图谱区域按四个章节主题组织知识点，避免直接铺开全量乱图。顶部搜索框仅在图谱总览中显示，输入知识点、概念或章节关键词后回车，可在总览图中定位并高亮相关节点。

### 教师端

教师端用于辅助教师按教材章节分析知识结构：

- `章节知识结构`：展示当前章节主题子图和核心知识点局部结构
- `关系统计与重点`：展示核心知识点、关系类型分布、教学要点提示和资料库推荐
- `三元组与证据`：展示知识关系三元组、来源教材、片段编号和证据文本
- 可调用本地 Ollama 生成较完整的教学要点说明，辅助教师备课和答辩演示

### 学生端

学生端用于展示基于知识图谱的学习辅助能力：

- `知识点查询`：搜索知识点，查看所属章节、相邻知识点、关系和教材证据
- `学习问答`：基于图谱三元组检索和教材证据，由本地 Ollama 组织自然语言回答
- `资源推荐`：根据当前知识点所属章节、相邻关系和预置资源库生成学习推荐说明
- `学习路径`：沿图谱关系生成连续学习链路，并以图谱方式辅助展示
- 学生可以将知识点标记为“已理解”，系统根据已理解知识点数量计算章节掌握进度，进度保存到 `outputs/kg/learning_progress.json`

## 6. 输出文件

主题集群构建后会生成：

- `outputs/kg/topic_clusters.json`
- `outputs/kg/topic_cluster_summary.md`

图谱构建结果包括：

- `outputs/kg/nodes.csv`
- `outputs/kg/edges.csv`
- `outputs/kg/kg_summary.json`
- `outputs/kg/kg_analysis_report.md`

学习进度文件：

- `outputs/kg/learning_progress.json`

该文件记录学生已标记为“已理解”的知识点，用于计算章节掌握进度。

## 7. 操作注意事项

- 页面左侧只有三个主入口：`图谱总览`、`教师端`、`学生端`。
- `图谱总览` 顶部搜索框只在总览页显示，输入知识点、概念或章节关键词后按回车，可在总览图中定位相关节点。
- `教师端` 中的三个按钮用于切换不同展示内容：`章节知识结构`、`关系统计与重点`、`三元组与证据`。
- `学生端` 中的四个按钮用于切换学习功能：`知识点查询`、`学习问答`、`资源推荐`、`学习路径`。
- 如果需要使用大模型问答、教学要点或推荐说明，请先启动本地 Ollama，并确保 `qwen3:1.7b` 模型可用。
- 如果本地 Ollama 未启动，系统仍可展示图谱、三元组、教材证据和规则推荐内容。
- 学生端的“已理解”标记会写入 `outputs/kg/learning_progress.json`，如果需要清空学习进度，可以删除该文件后重新启动页面。
- 导入新的三元组文件后，建议重新运行主题构建并刷新页面，确保图谱总览、教师端和学生端使用的是最新数据。
